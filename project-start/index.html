1. ก่อนกรอก: กติกาเริ่ม Project

ฟอร์มนี้ไม่ได้มีไว้เก็บ idea เฉยๆ แต่มีไว้ตัดสินใจว่า Project พร้อมจะเข้า process ต่อหรือยัง

Rule 1เริ่มจากปัญหา ไม่ใช่ solution

อย่าเริ่มด้วย “ทำ app/ทำ dashboard/ทำ AI” ให้บอกก่อนว่าปัญหาคืออะไร ใครเจอ และมีหลักฐานอะไร

Rule 2ต้องมี owner

ทุก project ต้องมีคนรับผิดชอบ business outcome, product scope และ final decision

Rule 3ต้องมี acceptance criteria

ถ้าวัดไม่ได้ว่าเสร็จและดีขึ้นจริง ยังไม่ควรเริ่ม development

3. Readiness Gate

        
Project พร้อมเข้า implementation ก็ต่อเมื่อช่อง critical ไม่ว่าง และ PM/Founder approve ว่า evidence, scope และ acceptance criteria เพียงพอ

        

          
Critical fieldทำไมสำคัญถ้ายังว่าง

          
            
Problemกันการ build solution ที่ไม่มีปัญหาจริงกลับไป discovery

            
User / customerกันการออกแบบกว้างเกินไปนิยาม ICP/user segment

            
Evidenceพิสูจน์ว่าปัญหาไม่ใช่ assumptionเก็บ interview/ticket/metric

            
Impactใช้จัด priority และ ROIประเมิน cost/frequency/severity

            
MVP + non-goalsกัน scope creepกลับไป product strategy

            
Acceptance criteriaรู้ว่าเสร็จจริงเมื่อไรเขียน PRD/test plan ก่อน

          
        

        
No brief, no build: ถ้า form นี้ยังไม่ชัด ห้ามเริ่ม implementation ให้ส่งกลับไป Discovery, PRD หรือ Architecture ตามช่องที่ขาด
