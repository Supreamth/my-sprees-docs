Project Overview Page Template

ทุก Project ต้องมีหน้า Overview ให้คนและ AI ติดตามงานเดียวกัน

ใช้เป็น dashboard กลางของ project รวมสถานะ phase, source-of-truth links, decisions, risks, blockers และ next step task/prompt ที่ copy ไปสั่ง model ถัดไปได้เสมอ

Created whenNew project starts

RuleNo hidden status

1. Purpose

หน้า Overview คือหน้าแรกของ project ที่ทุกคนเปิดดูได้ทันทีว่า project คืออะไร อยู่ขั้นไหน ต้องทำอะไรต่อ และต้องสั่ง model ด้วย prompt อะไร

Trackingติดตาม process ทั้งหมด

Project Start → Discovery → PRD → UX → Architecture → Build → QA → Release → Feedback

Continuityต่อเนื่องระหว่าง model

รวม source-of-truth และ model handoff เพื่อให้ model ใหม่ทำงานต่อได้ทันที

Next actionมี prompt พร้อมสั่งต่อ

หน้า overview ต้องมี next human instruction, AI handoff prompt และ reviewer prompt เสมอ

2. Required sections

ทุก Project Overview Page ต้องมี section เหล่านี้อย่างน้อย

SectionWhat it showsWhy it matters
Project headername, one-line summary, owner, phase, status, last updatedรู้ทันทีว่า project คืออะไรและใครรับผิดชอบ
Snapshot cardsphase, risk, next gate, next action, recommended modelอ่านเร็วสำหรับ founder/PM
Process timelinestatus ของทุก phaseเห็นว่ากระบวนการไม่ข้าม gate
Source of truthlinks ไป PRD, requirements, architecture, current-state, decisionsลด token และ context drift
Current statecompleted, in progress, blocked, latest verificationmodel ใหม่ไม่ต้องเดา state
Next step task/prompthuman instruction, AI prompt, reviewer promptสั่งงานต่อได้ทันที
Open questionsquestion, owner, due, impactblocker มีเจ้าของ
Decisions / Risks / Activitydecision log, risk table, chronological updatesตรวจย้อนหลังได้

3. Process timeline states

ใช้ status เดียวกันทุก project เพื่อให้เทียบข้าม project ได้

Statenot started

ยังไม่เริ่ม phase นี้

Statein progress

กำลังทำ มี owner และ next action

Stateblocked

มี blocker ต้องมี owner และ impact

Stateready for review

artifact พร้อมให้ reviewer/human ตรวจ

Statecompleted

ผ่าน gate แล้ว มี evidence

Stateskipped with reason

ข้ามได้ แต่ต้องบอกเหตุผล

4. Next step task / prompt

หน้า Overview ต้องมี prompt ที่ copy ได้เสมอ เพื่อให้ human หรือ model ถัดไปทำงานต่อได้โดยไม่ต้องถามใหม่

You are taking over this project as: [Role]

Read first:
- [Project overview URL]
- [Current state file]
- [Model handoff file]
- [Relevant source-of-truth docs]

Current objective:
[objective]

Current phase:
[phase]

Do this next:
1. [next action]
2. [verification/update]

Do not do:
- Do not skip the current gate
- Do not invent missing requirements
- Do not overwrite source-of-truth decisions without updating decision log

Expected output:
- Summary
- Updated artifacts
- Decisions needed
- Verification performed
- Updated next-step prompt

5. Operating rule

เพื่อไม่ให้ project หลุด context ทุก meaningful model run ต้องอัปเดต overview หรือเสนอ update กลับมา

Rule: Every meaningful model run must either update the Project Overview Page or return a proposed update for the page.
No hidden status: ถ้าไม่มี visible next step และ copyable next prompt ห้ามส่งต่อให้ model ถัดไป เพราะจะเกิด context gap
